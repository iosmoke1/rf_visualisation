# Random Forest Visualisation

Интерактивная web-визуализация алгоритма Random Forest для задачи бинарной классификации на двумерных данных. Проект сделан как учебная песочница: пользователь меняет датасет и параметры леса, обучает модель по шагам и сразу видит, как меняются граница решений, деревья, метрики и bootstrap-выборки.

- Приложение: https://iosmoke1.github.io/rf_visualisation/
- Репозиторий: https://github.com/iosmoke1/rf_visualisation
- Технологии: HTML, CSS, JavaScript, Canvas, Jupyter Notebook, scikit-learn

## Задача проекта

Цель проекта - показать механику Random Forest так, чтобы ее можно было объяснять на интерактивной лекции. Приложение связывает несколько уровней работы модели:

- исходные точки train/test;
- bootstrap-выборку отдельного дерева;
- split-линии внутри дерева;
- пошаговое добавление деревьев в лес;
- итоговую границу решений;
- метрики качества и важность признаков.

## Возможности приложения

- генерация датасетов: две луны, кольца, XOR, кластеры;
- настройка параметров данных: размер выборки, шум, доля test, seed;
- настройка Random Forest: число деревьев, максимальная глубина, `min_samples_split`, `min_samples_leaf`, bootstrap-доля, `max_features`;
- обучение леса по одному дереву вперед и назад;
- автоматическая анимация обучения леса;
- режим показа split-линий по узлам во время анимации;
- просмотр первых деревьев в виде вложенной структуры условий;
- подсветка bootstrap-выборки выбранного дерева;
- визуализация границы решений на `canvas`;
- метрики train/test: accuracy, precision, recall, F-метрика, ROC-AUC, PR-AUC;
- confusion matrix в формате TP, TN, FP, FN;
- важность признаков по суммарному уменьшению Gini impurity;
- отдельная Jupyter-тетрадка с примером обучения `RandomForestClassifier` из `sklearn`.

### C4: Context

Пользователь работает с приложением в браузере. Он выбирает датасет, задает параметры Random Forest, запускает обучение и анализирует результат по графику, деревьям и метрикам. Внешних серверов, баз данных и API у приложения нет.

```text
Пользователь -> Браузерное приложение -> GitHub Pages
```

### C4: Container

Внутри проекта есть один основной контейнер - статический сайт.

```text
Static Web App
├── HTML-разметка интерфейса
├── CSS-стили
└── JavaScript-модель и визуализация
```

GitHub Pages отвечает только за отдачу файлов. Обучение модели, расчет метрик и отрисовка выполняются в браузере.

### C4: Components

Основной файл `app.js` можно условно разделить на несколько частей:

- генератор датасетов - создает точки для выбранной учебной задачи;
- `DecisionTree` - строит одно решающее дерево по критерию Gini impurity;
- `RandomForest` - обучает набор деревьев на bootstrap-выборках и усредняет их предсказания;
- блок метрик - считает accuracy, precision, recall, F-метрику, ROC-AUC, PR-AUC и confusion matrix;
- блок визуализации - рисует точки, границу решений, split-линии и подсветку bootstrap-выборки;
- блок интерфейса - синхронизирует ползунки, кнопки, меню обучения, список деревьев и статистику.

## Как устроено обучение

Для каждого дерева создается bootstrap-выборка из train-части датасета. Дерево рекурсивно ищет лучший split по признакам `x` и `y`, максимизируя уменьшение Gini impurity. В листе сохраняется доля объектов положительного класса, а лес усредняет вероятности по всем обученным деревьям.

Граница решений строится по сетке точек: для каждой ячейки считается вероятность класса `1`, после чего фон окрашивается в цвет соответствующего класса. При добавлении или удалении дерева граница пересчитывается, поэтому видно, как ансамбль постепенно стабилизирует предсказание.

## Jupyter Notebook

Файл `random_forest_demo.ipynb` нужен как отдельная демонстрация работы модели через готовую библиотеку. В нем используется `sklearn.ensemble.RandomForestClassifier`: обучение, предсказание, confusion matrix, метрики качества, визуализация границы решений, важность признаков и просмотр одного дерева из леса.

Ноутбук дополняет web-приложение: сайт показывает механику алгоритма интерактивно, а тетрадка показывает, как ту же модель обычно применяют в Python.

## Отличие от аналогов

В качестве ориентиров рассматривались MLU-Explain Random Forest и TensorFlow Playground.

MLU-Explain хорошо объясняет bagging, случайный выбор признаков и голосование деревьев, но в нем меньше свободы для изменения датасета, гиперпараметров, метрик и пошагового обучения.

TensorFlow Playground дает сильный пример интерактивности и визуализации границы решений, но посвящен нейронным сетям, а не Random Forest. В нем нет деревьев, bootstrap-выборок, OOB-идеи и важности признаков леса.

В этом проекте акцент сделан именно на Random Forest: пользователь видит не только итоговую границу решений, но и связь между деревьями, bootstrap-выборками, split-линиями, голосованием и метриками.

## Ограничения

- приложение рассматривает только бинарную классификацию;
- признаки двумерные, потому что это удобно для наглядной визуализации;
- реализация Random Forest написана для учебной демонстрации и не заменяет `sklearn`;
- большие леса можно обучать, но для просмотра деревьев показывается ограниченное число карточек.
